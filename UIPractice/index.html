Getting Started with NoesisGUI in Unity

Your project has been generated! This document will describe how to run the project in both Blend and Unity.

Project Structure

Depending on your choice during creation, the project folder will either be a child of the solution folder, or the same as the solution folder.

The project folder contains an Assets and Blend folder:

  • Assets: All resources which are shared between Unity and Blend, including XAML, Code-Behind classes, fonts, and textures, are placed in this folder.
  • Blend: This folder contains resources which are required for Blend, which should not be exposed to Unity/NoesisGUI.

Note: The solution has been renamed, adding "-blend", to avoid conflicts with the solution which Unity generates.

Visual Studio and Blend

The project is ready to run in Visual Studio and Blend, simply click Start on the top bar, or press F5.

Unity

Before opening the project in Unity, you'll need to install the NoesisGUI Unity Package.

Please follow these steps:

  1. Download the latest NoesisGUI Unity Package.
  2. Extract the NoesisGUI Unity Package zip archive to the LocalPackages\com.noesis.noesisgui\ directory, which is located in the project folder. The package.json file should be located in the com.noesis.noesisgui folder.
  3. Open Unity Hub, click Add, and select the project folder. This is the folder which contains the Assets and LocalPackages folders.
  4. In the Unity Hub project list, select a Unity version for the project.
  5. Open the project, and select Continue in the upgrade popup.

To load the UI in Unity, open the application Scene in the Assets folder and click Play.
